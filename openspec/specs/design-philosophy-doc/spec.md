## 新增需求

### 需求:设计理念页面
文档站必须提供 `/guide/design-philosophy` 页面，阐述 Sora 框架的四个核心设计理念：渐进式基础设施抽象、MVP 到生产无缝演进、CLI 驱动的开发体验、开箱即用的工程基础设施。

#### 场景:技术决策者评估框架
- **当** 技术决策者访问设计理念页面
- **那么** 页面必须清晰展示四个核心价值点，每个要点包含概述阐述和具体功能支撑

#### 场景:开发者理解渐进式抽象
- **当** 开发者阅读"渐进式基础设施抽象"章节
- **那么** 页面必须展示 RamDiscovery 到 ETCDDiscovery 的代码切换示例，以及 RPC 传输协议和编解码器的可替换性

#### 场景:开发者理解演进路径
- **当** 开发者阅读"从 MVP 到生产"章节
- **那么** 页面必须展示三阶段演进路径（单进程 → 多服务 → 集群），并强调业务代码无需修改

#### 场景:开发者了解 CLI 能力
- **当** 开发者阅读"CLI 驱动的开发体验"章节
- **那么** 页面必须列出核心 CLI 命令及其功能，展示脚手架如何消除样板工作

#### 场景:开发者了解工程基础设施
- **当** 开发者阅读"开箱即用的工程基础设施"章节
- **那么** 页面必须以表格形式列出每个基础设施功能的一句话描述和对应文档链接（类型校验、分布式追踪、结构化日志、上下文传播、配置管理）

### 需求:首页定位更新
首页 tagline 必须从"高性能微服务框架"更新为"渐进式微服务框架"。

#### 场景:访客看到新 tagline
- **当** 访客打开文档站首页
- **那么** hero 区域的 text 必须显示"渐进式微服务框架"

### 需求:首页设计理念入口
首页 hero actions 必须包含"设计理念"按钮，链接到 `/guide/design-philosophy`。

#### 场景:从首页导航到设计理念
- **当** 访客点击首页"设计理念"按钮
- **那么** 必须跳转到 `/guide/design-philosophy` 页面

### 需求:首页 features 微调
首页 features 四条条目的 details 文本必须微调以呼应"渐进式"设计理念。

#### 场景:features 呼应渐进式理念
- **当** 访客阅读首页 features 区域
- **那么** 四条 feature 的 details 描述必须体现"可替换"、"可演进"、"渐进式"等核心理念

### 需求:指南侧边栏新增条目
指南侧边栏必须包含"设计理念"条目，链接到 `/guide/design-philosophy`。

#### 场景:从侧边栏导航到设计理念
- **当** 用户在指南区域浏览页面
- **那么** 侧边栏必须显示"设计理念"条目，位于"快速开始"之后
