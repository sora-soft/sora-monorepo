## 上下文

Sora 文档站（VitePress）位于 `docs/` 目录，当前首页 tagline 为"高性能微服务框架"。指南部分有快速开始、微服务、使用组件、文档与类型导出四个页面。缺少一个阐述设计理念和选型理由的页面。

现有文档中 `docs/microservice/discovery.md` 已经描述了 RamDiscovery → ETCDDiscovery 的演进路径，但没有从全局视角总结这个设计哲学。本变更将这一理念提炼为独立页面，并调整首页定位。

## 目标 / 非目标

**目标：**
- 新增设计理念页面，清晰传达 Sora 的四个核心价值：渐进式基础设施、MVP 到生产演进、CLI 开发体验、开箱即用的工程基础设施
- 将首页定位从"高性能"调整为"渐进式"
- 为技术决策者提供评估依据，为开发者提供理解框架设计哲学的入口

**非目标：**
- 不修改除 index.md 和 config.ts 之外的现有文档内容
- 不做竞品对比
- 不修改框架代码或 API
- 不涉及国际化（保持中文）

## 决策

### 页面结构采用四段式

四个核心要点各自独立成章，每章包含：一句话概述、一段阐述、必要的代码示例或图表。按以下顺序排列：

1. **渐进式基础设施抽象** — 最核心的差异化，放第一
2. **从 MVP 到生产** — 渐进式的自然延伸，展示演进路径
3. **CLI 驱动的开发体验** — 从设计哲学转向实际使用体验
4. **开箱即用的工程基础设施** — 功能完备性的收尾

### 首页 tagline 改为"渐进式微服务框架"

"高性能"是结果，"渐进式"是设计理念。渐进式更能传达 Sora 的独特价值，且与实际使用体验一致。

### Features 微调

保持四个 feature 条目不变（RPC 通信、服务发现、组件系统、类型安全），微调 details 措辞以呼应"渐进式"理念。

### 不引入新的代码示例文件

页面中的代码示例直接内嵌在 markdown 中，使用已有的 RamDiscovery/ETCDDiscovery 切换示例和 CLI 命令示例。

## 风险 / 权衡

- [内容可能偏营销化] → 保持技术实质，每个要点配具体功能表格和文档链接，避免空洞表述
- [Features 微调可能影响现有理解] → 仅微调 details 文本，不改变 feature 的 title，保持一致性
